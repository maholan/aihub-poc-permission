5 แนวคิดหลัก

🏛️

ผูกกับตำแหน่ง

โยกย้าย/เลื่อนระดับ ต.ค. → แก้แค่ตำแหน่ง/กอง สิทธิ์ปรับทั้งหมดอัตโนมัติ

⊞

2 แกน (AND)

Domain (เรื่อง) × ชั้นความลับ — ต้องผ่านทั้งคู่ ขาดอย่างใดอย่างหนึ่งคือไม่เห็น

🧩

กลุ่มย่อย

grant domain เพิ่มข้ามกอง เช่น คกก.ความมั่นคง เห็นข้อมูลวิศวกรรม

🌲

Tree + Inherit

collection = โฟลเดอร์+ไฟล์ ซ้อนกัน — ชั้นความลับไหลลง (โฟลเดอร์ลับ→ไฟล์ลับ)

🔐

Exception เฉพาะกลุ่ม/คน

ตั้งที่โฟลเดอร์ได้ — เฉพาะคนที่กำหนด (แม้ผู้บริหารสูงสุดก็ถูกกันได้)

⏱️

ตำแหน่งชั่วคราว

ช่วยราชการ = เปิดสิทธิ์ชั่วคราว มีวันหมดอายุ

ใครสร้าง Collection ได้ — และระดับไหน

👥

ใครสร้างได้

role ที่มีสิทธิ์ “จัดการ Collection” — default ผู้จัดการขึ้นไป (เจ้าหน้าที่สร้างไม่ได้)

🗂️

เพดาน Domain

สร้างได้เฉพาะ domain ที่ตัวเองเข้าถึง — ผจก.HR สร้าง HR ได้, การเงินไม่ได้

🎚️

เพดานชั้นความลับ

ตั้งชั้นได้ ไม่เกิน clearance ตัวเอง — clearance ลับ → ตั้งได้ถึงลับ ไม่ถึงลับที่สุด

สร้างได้ ⟺ มีสิทธิ์ ManageCollection และ domain ∈ สิทธิ์ตัวเอง และ ชั้นที่ตั้ง ≤ clearance
ผู้สร้าง = owner · สร้างซ้อนได้ถ้าชั้นลูก ≥ floor แม่ · ผู้บริหาร/ทีมกลางข้าม domain+ทุกชั้นได้

ประเด็นที่ต้องเคาะในที่ประชุม

  • 1. Clearance มาจากไหน — ผูก "ระดับตำแหน่ง" แบบ global หรือ per-domain (เป็น ผจก. กอง HR ไม่ควรเห็นลับของกองการเงิน)? — แนะ per-domain
  • 2. รักษาการ vs ช่วยราชการ — ต่างกันไหม? ต้อง delegation แบบ time-boxed + audit
  • 3. Break-glass — เข้าถึง "ลับที่สุด" ฉุกเฉิน → log พิเศษ + แจ้งเจ้าของ
  • 4. เอกสารข้ามกอง — ติดหลาย domain → ใช้ most-restrictive?
  • 5. Sync ผังองค์กร — ดึงจาก HR/AD แบบ real-time หรือ batch? ช่วงเปลี่ยนผ่านจัดการยังไง
  • 6. ชั้นความลับ — ใช้มาตรฐาน 3 ชั้น (ลับ/ลับมาก/ลับที่สุด) + ทั่วไป ตรงกับ กปน. ไหม หรือมี label ภายในเพิ่ม
