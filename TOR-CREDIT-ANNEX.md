# เอกสารแนบ — หลักเกณฑ์และวิธีการคำนวณเครดิต (Credit) สำหรับการใช้งานโมเดล AI

> ใช้ประกอบ TOR ข้อกำหนดด้านการบริหารจัดการ Token และเครดิต — เอกสารหลักระบุโควตาเป็น **จำนวนเครดิต** เท่านั้น; วิธีการคำนวณและการแปลงเป็นไปตามเอกสารแนบฉบับนี้

---

## 1. นิยาม

1.1 **"เครดิต (Credit)"** หมายถึง หน่วยกลางที่ใช้วัดปริมาณการใช้ทรัพยากรประมวลผลของโมเดล AI ในระบบ เพื่อการจัดสรร ควบคุม หักลด และรายงานการใช้งาน โดยเอกสารฉบับนี้ไม่ผูกเครดิตกับมูลค่าสกุลเงินใด

1.2 **"Token"** หมายถึง หน่วยย่อยของข้อความที่โมเดล AI ใช้ในการประมวลผล ตามนิยามของผู้พัฒนาโมเดลแต่ละราย

1.3 **"Input Token"** หมายถึง Token ขาเข้าที่ส่งให้โมเดลประมวลผล (รวมคำสั่งระบบ บริบท เอกสารอ้างอิง และคำถามของผู้ใช้)

1.4 **"Output Token"** หมายถึง Token ขาออกที่โมเดลสร้างขึ้นเป็นคำตอบ

1.5 **"MTok"** หมายถึง 1,000,000 Token

1.6 **"อัตราเครดิต (Credit Rate)"** หมายถึง จำนวนเครดิตที่หักต่อ 1 MTok โดยกำหนด **แยกระหว่าง Input และ Output เป็นรายโมเดล**

## 2. หลักการ

2.1 ระบบใช้ **เครดิตกลางชุดเดียว (Central Credit Pool)** — การเรียกใช้โมเดลทุกรายการหักจากเครดิตกลางผ่านอัตราเครดิตของโมเดลนั้น

2.2 โควตาการใช้งานในเอกสารหลักกำหนดเป็น **จำนวนเครดิตต่อปี** และจัดสรรย่อยได้หลายระดับ (องค์กร / หน่วยงาน / ผู้ใช้ / Agent) เป็นรายเดือน โดยผลรวมการจัดสรรระดับล่างต้องไม่เกินระดับบน

2.3 มูลค่าทางการเงินของเครดิต (หากมี) ให้เป็นไปตามที่กำหนดในสัญญาหรือเอกสารด้านราคาแยกต่างหาก **ไม่ปรากฏในเอกสารฉบับนี้และเอกสารหลัก**

2.4 อัตราเครดิตทุกรายการต้อง **กำหนดค่าได้ผ่านหน้าจอผู้ดูแลระบบ** (Configurable) ห้ามฝังค่าตายตัวในโปรแกรม และระบบต้องเก็บประวัติอัตราทุกช่วงเวลา (Rate History พร้อมวันที่มีผล) เพื่อการตรวจสอบย้อนหลังและการคิดยอดตามอัตราที่มีผลจริง ณ เวลาใช้งาน

## 3. สูตรการคำนวณเครดิตต่อการเรียกใช้ (Request)

```
เครดิตที่หัก = ( Input Token ÷ 1,000,000 × อัตราเครดิต Input ของโมเดล )
            + ( Output Token ÷ 1,000,000 × อัตราเครดิต Output ของโมเดล )
```

3.1 จำนวน Token ให้ยึดตามค่าการใช้งานจริง (Usage) ที่ผู้ให้บริการโมเดลรายงานกลับต่อการเรียกใช้แต่ละครั้ง

3.2 การคำนวณภายในเก็บทศนิยมไม่น้อยกว่า 4 ตำแหน่ง การแสดงผลปัดเศษ 2 ตำแหน่ง (ปัดเศษครึ่งขึ้น)

3.3 การหักเครดิตกระทำ **ทันทีต่อการเรียกใช้** โดยแยกบันทึกส่วน Input และ Output (สอดคล้องข้อกำหนดการคำนวณต้นทุนแยก Input/Output Token)

3.4 การเรียกใช้ที่ล้มเหลวโดยไม่มีการประมวลผลของโมเดล (เช่น ถูกปฏิเสธก่อนส่งเข้าโมเดล) ไม่หักเครดิต

## 4. ที่มาของอัตราเครดิต (Rate Derivation)

4.1 อัตราเครดิตของโมเดลแบบบริการภายนอก (Public Model) คำนวณจาก

```
อัตราเครดิต (ต่อ 1 MTok) = P × FX × K
```

| ตัวแปร | ความหมาย | แหล่งอ้างอิง |
|---|---|---|
| **P** | ราคาประกาศทางการของผู้ให้บริการโมเดล (ต่อ 1 MTok แยก Input/Output) | หน้าราคาทางการของผู้ให้บริการ ณ วันที่กำหนดอัตรา |
| **FX** | อัตราอ้างอิงการแปลงสกุลเงิน | อัตราอ้างอิงธนาคารแห่งประเทศไทย หรือค่าคงที่รายรอบตามที่คู่สัญญาตกลง |
| **K** | ค่าสัมประสิทธิ์ดำเนินการ ครอบคลุมค่าบริหารระบบ ค่าดำเนินการ และความเสี่ยงจากความผันผวนของราคา/อัตราแลกเปลี่ยน | ตามที่กำหนดในสัญญา (ค่าเดียวใช้ทุกโมเดล เว้นแต่ระบุเป็นอย่างอื่น) |

4.2 อัตราเครดิตของโมเดลติดตั้งภายในองค์กร (Local Model) ให้กำหนดเป็น **อัตรากำหนด (Nominal Rate)** เพื่อการควบคุมปริมาณการใช้และการเปรียบเทียบข้ามโมเดล โดยไม่อ้างอิงราคาผู้ให้บริการภายนอก ทั้งนี้ตามที่คณะกรรมการกำกับ (ตามข้อ 5.2) เห็นชอบ

4.3 อัตราเครดิตที่มีผลใช้จริงทุกโมเดล ต้องแสดงในหน้าจอระบบให้ผู้มีสิทธิ์ตรวจสอบได้ตลอดเวลา

## 5. การปรับปรุงอัตราเครดิต

5.1 เมื่อผู้ให้บริการโมเดลปรับราคาประกาศ (P) หรืออัตราอ้างอิง (FX) เปลี่ยนแปลงเกินกรอบที่ตกลง ผู้รับจ้างจัดทำอัตราเครดิตใหม่พร้อมหลักฐานที่มา เสนอต่อผู้ว่าจ้างล่วงหน้าไม่น้อยกว่า **15 วันทำการ** ก่อนวันมีผล

5.2 การปรับอัตราต้องได้รับความเห็นชอบจากผู้ว่าจ้างหรือคณะกรรมการที่ผู้ว่าจ้างมอบหมาย และมีผล **ไปข้างหน้าเท่านั้น** (ไม่ปรับย้อนหลังกับเครดิตที่หักไปแล้ว)

5.3 ระบบบันทึกการปรับอัตราทุกครั้ง (ผู้กระทำ วันเวลา อัตราเดิม อัตราใหม่ วันมีผล เหตุผล) ไว้ใน Audit Log

5.4 การเพิ่ม/ถอดโมเดลออกจากระบบ ให้ดำเนินการตามกระบวนการเดียวกับข้อ 5.1–5.3

## 6. การหัก การควบคุม และการรายงาน

6.1 ระบบแสดงยอดเครดิตคงเหลือของผู้ใช้/หน่วยงาน และแจ้งเตือนเมื่อใช้ถึง **ร้อยละ 80** ของโควตาที่ได้รับ

6.2 การเติมหรือขอเพิ่มเครดิตระหว่างรอบ กระทำได้ทั้งโดยผู้ดูแลระบบโดยตรง และผ่านกระบวนการขออนุมัติ (Approval Workflow)

6.3 ระบบสรุปยอดการใช้เครดิตเมื่อสิ้นรอบ (Billing Cycle) และจัดทำรายงานการใช้ระดับ Session / ผู้ใช้ / หน่วยงาน แยกรายโมเดล และแยก Input/Output Token

6.4 การเรียกใช้ผ่าน Agent ให้หักและรายงานแยกต่อ Agent แต่ละตัว

6.5 กรณีเปิดใช้ส่วนลดของผู้ให้บริการ (เช่น การประมวลผลแบบกลุ่ม Batch หรือ Cached Input) ให้หักเครดิตตามอัตราที่สะท้อนส่วนลดจริง และแสดงในรายงาน

## 7. ตัวอย่างการคำนวณ (ตัวเลขสมมติเพื่อประกอบความเข้าใจ)

กำหนดให้โมเดล A มีอัตราเครดิต: Input = 215 เครดิต/MTok, Output = 1,073 เครดิต/MTok

ผู้ใช้ถามคำถาม 1 ครั้ง ใช้ Input 4,700 Token และ Output 500 Token:

```
เครดิตที่หัก = (4,700 ÷ 1,000,000 × 215) + (500 ÷ 1,000,000 × 1,073)
            = 1.0105 + 0.5365
            = 1.5470 ≈ 1.55 เครดิต
```

## 8. ตารางอัตราเครดิตอ้างอิง ณ วันเริ่มให้บริการ

> ตัวอย่างประกอบ — อัตราที่มีผลจริงให้ยึดตามประกาศในระบบ (ข้อ 4.3) และประวัติอัตรา (ข้อ 2.4)

| กลุ่มโมเดล | อัตราเครดิต Input (ต่อ MTok) | อัตราเครดิต Output (ต่อ MTok) |
|---|--:|--:|
| โมเดลความสามารถสูงสุด (Frontier) | 429 | 2,145 |
| โมเดลความสามารถสูง (Advanced) | 215 | 1,073 |
| โมเดลมาตรฐาน (Standard) | 129 | 644 |
| โมเดลเร็ว/ประหยัด (Fast) | 43 | 215 |
| โมเดลติดตั้งภายใน (Local — Nominal) | 13 | 64 |

หมายเหตุ: การจับคู่โมเดลจริงเข้ากลุ่ม และอัตรารายโมเดล ให้เป็นไปตามประกาศในระบบ ซึ่งผ่านความเห็นชอบตามข้อ 5
